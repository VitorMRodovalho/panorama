### Decision tree for contributors (for the ADR appendix)

```
  Is there a tenantId available at the call site?
  ├─ NO ──────────────────────────────→ runAsSuperAdmin
  │   (login, OIDC callback, PAT hash lookup, discovery, token-based invitation accept)
  │
  └─ YES
      │
      Does the operation touch data from OTHER tenants?
      ├─ YES ───────────────────────────→ runAsSuperAdmin
      │   (cluster-wide sweep, outbox claim, cross-tenant import,
      │    audit chain append, tenant-creation)
      │
      └─ NO
          │
          Does the operation need the Owner-check trigger bypass?
          ├─ YES ─────────────────────────→ runAsSuperAdmin
          │   (break-glass nominateOwner)
          │
          └─ NO ──────────────────────────→ runInTenant(tenantId)
```

**One-liner for the team:** _"If you have a `tenantId` and you're only reading/writing that tenant's rows, you MUST use `runInTenant`."_

### Proposed CI gate

File-level allowlist at `.github/superadmin-allowlist.txt`. A script `scripts/check-superadmin-allowlist.sh` fails CI when any file NOT on the allowlist calls `runAsSuperAdmin`. Full script and allowlist contents are in the Wave 2 GitHub issues. The allowlist covers the 30 JUSTIFIED sites; all 23 LAZINESS sites must migrate to `runInTenant` or fail the gate.

### PrismaService weaknesses that encourage misuse

1. `runInTenant` requires passing `tenantId` explicitly; `runAsSuperAdmin` is "free." An `AsyncLocalStorage`-backed `currentTenantId()` reader would make `runInTenant()` (no arg) the path of least resistance.
2. `reason` parameter is unvalidated free-text — structured enum or min-length would improve log forensics.
3. No call-site telemetry (`caller` field from stack) in the super-admin warn log.
4. Dev/test same-URL bypass means developers never discover missing `runInTenant` locally.

## Wave 2b findings — notification bus end-to-end trace

**Scope:** every `panorama.*.*` event-type string in the codebase, every `enqueueWithin` call site, `NOTIFICATION_PAYLOAD_SCHEMAS` registry, channel handlers, plugin-sdk event union, ADR references.

### Event registry classification (76 events)

| Classification | Count | Meaning |
|---|---:|---|
| ✅ FULLY WIRED | 3 | Emitted through bus, schema'd, at least one channel handles it |
| AUDIT-ONLY | 49 | Emitted via `audit.record` / `audit.recordWithin`, never enters bus |
| 🟠 GHOST (SDK) | 4 | Declared in `PanoramaEventName` but never emitted |
| 🟠 GHOST (ADR-0016) | ~12 | Documented in ADR-0016 but maintenance service not built |
| 🟡 BUS UNREGISTERED | 1 | ADR-committed bus event (`checked_in_with_damage`) not emitted |
| Dispatcher-internal | 7 | Emitted by dispatcher itself (enqueued, dispatched, dead, rescued, etc.) |

**Only 3 of 76 events (4%) are fully wired.** The bus is architecturally ready but operationally barely populated.

### Top findings

| ID | Severity | Title |
|---|---|---|
| NOTIF-01 | critical | `panorama.reservation.checked_in_with_damage` — ADR-committed but never emitted (reconfirms Wave 1 ARCH-15) |
| NOTIF-02 | high | Plugin SDK `PanoramaEventName` is hand-maintained; 4 of 7 events are ghosts |
| NOTIF-04 | high | Schema registry has only 3 entries; ADR-0011 + ADR-0016 commit to 10+ |
| NOTIF-05 | high | Tamper trigger breaks the audit hash chain (reconfirms DATA-03 at source) |
| NOTIF-07 | high | `enqueueWithin`'s `eventType` is `string` — no compile-time check |
| NOTIF-03 | medium | `panorama.reservation.created` in SDK but audit-only |
| NOTIF-06 | medium | Tamper trigger payload shape ≠ what the test asserts |
| NOTIF-08 | medium | `panorama.inspection.completed` has no e2e bus test |
| NOTIF-09 | low | Channel `name` inconsistency (`email` vs `inspection-outcome-email`) |
| NOTIF-10 | info | Zero-handler events silently DISPATCHED — no WARN log |

### Proposed registry discipline

**Principle:** `NOTIFICATION_PAYLOAD_SCHEMAS` is the single source of truth.

1. **Type-safe emit.** Change `NotificationEventInput.eventType` from `string` to `keyof typeof NOTIFICATION_PAYLOAD_SCHEMAS`. Every unregistered emit becomes a compile error.
2. **SDK derives from core.** `PanoramaEventName` in the plugin-sdk imports the derived type (or shared-types package if dependency direction prohibits direct import).
3. **CI drift gate.** A `scripts/check-notification-registry.ts` that fails when:
   - An `enqueueWithin` call references a type not in the registry.
   - A registered type has no emit site (dead schemas).
   - A registered type has no channel handler AND is not on an explicit "bus-only, no handler yet" allowlist.
4. **Audit vs bus naming.** Generate `docs/en/event-catalog.md` listing every event with a `bus: yes/no` column, so audit-only events (~49) are explicitly documented as such.

### Artifacts delivered

A runnable baseline script is committed at [`apps/core-api/prisma/perf/baseline.sql`](../../apps/core-api/prisma/perf/baseline.sql) (generated by Wave 2c). It:
1. Seeds a synthetic tenant with realistic cardinality (200 assets, 10k reservations, 5k inspections, 2k notifications, 500 invitations) — only if absent.
2. Runs `EXPLAIN (ANALYZE, BUFFERS, FORMAT TEXT)` on all 10 queries.
3. Writes output to a timestamped file under `perf/`.

Usage: `psql -U panorama -d panorama -f apps/core-api/prisma/perf/baseline.sql -o apps/core-api/prisma/perf/baseline-$(date +%Y%m%d-%H%M%S).txt` — commit the output file so reviewers can diff regressions.

A `pg_stat_statements` detection snippet is also embedded in the perf folder for use as a scheduled maintenance task surfacing queries exceeding latency/frequency/fan-out thresholds.

Two columns (`AuditEvent.metadata`, `Asset.customFields`) should be monitored for future GIN needs if admin-search or custom-field-search features land.

## Wave 2d findings — CI hardening (concrete diffs)

Every finding in this wave ships as a concrete diff or new file the maintainer can paste into a PR. Full text lives in the corresponding GitHub issues; summary here.

### A. ESLint for `apps/core-api`

ESLint 9 flat-config (`eslint.config.mjs`) with `typescript-eslint recommendedTypeChecked`. Starts permissive (`no-explicit-any`, `no-unsafe-*` as `warn`); errors only on the real-bug catches (`no-floating-promises`, `no-misused-promises`, `await-thenable`). Replaces the echo stub. `--max-warnings=0` makes every warning a CI fail → forces either fix or justified `eslint-disable-next-line`.

### B. ESLint for `apps/web`

ESLint 8 legacy `.eslintrc.cjs` pinned to match Next 14.2's `eslint-config-next`. `next lint --dir src --max-warnings=0`. Two ESLint majors coexist via pnpm workspace isolation.

### C. Trivy that actually blocks

Two-pass pattern: (1) SARIF upload step with `exit-code: 0` so the Security tab always updates; (2) blocking gate step with `exit-code: 1 --severity HIGH,CRITICAL --ignore-unfixed`. Pinned action to `@0.24.0` (was `@master`). Runs on PRs now, not just push. New `.trivyignore` scaffold with a mandatory-header template for accepted advisories (reviewer, re-review date, rationale, upstream-fix link).

### D. `ensure-community-complete` as a TS script

Replaces the `grep && exit 1 || exit 0` pipeline (which masks grep's error exit 2) with `scripts/ensure-community-complete.ts`. Uses explicit regex patterns for scoped (`@panorama/enterprise-*`) and unscoped (`panorama-enterprise`) naming. Scans TS/TSX/JSON sources with an allowlist-of-paths for this-script-itself. Structured exit codes (0/1/2).

### E. Coverage reporting

Adds `@vitest/coverage-v8`, `test:coverage` script, `coverage.thresholds` block with honest baseline (50/50/45/40). Publishes a markdown table to `$GITHUB_STEP_SUMMARY` and uploads the full report as a 14-day artifact. **Decision: GitHub summary, not Codecov** — rationale in the Wave 2d annex.

### F. Migration convention checker

`scripts/check-migration-conventions.ts` enforces every migration folder contains `migration.sql`, `ROLLBACK.md`, and exactly one of `rls.sql` or `NO_RLS.md` (with length-floor for skip rationale). Ships three new `NO_RLS.md` files for migrations 0002, 0003, and 0013 with justified skip rationale — closes Wave 1 ARCH-11.

### G. Drop aspirational `e2e` task

Removes `e2e` from `turbo.json` and root `package.json`. Comments out `pnpm e2e` in `CONTRIBUTING.md`. Fails fast (`turbo: task "e2e" not found`) instead of silently passing. Re-add when the first Playwright spec ships.

### H. Pin gitleaks action

`gitleaks/gitleaks-action@v2` → `@v2.3.9`. Three-line diff. Improves OSSF Scorecard pinned-dependencies score.
